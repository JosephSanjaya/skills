# Groovy DSL vs. Kotlin DSL (KTS) Task Syntax (Gradle 9.5.1+)

Gradle fully supports both Groovy DSL (`build.gradle`) and Kotlin DSL (`build.gradle.kts`). Kotlin DSL provides static typing, better IDE auto-completion, and refactoring support, but has stricter syntax rules.

---

## 1. Syntax Comparison Matrix

| Scenario | Groovy DSL (`build.gradle`) | Kotlin DSL (`build.gradle.kts`) |
|---|---|---|
| **Simple Lazy Task** | `tasks.register('hello') { doLast { println 'Hi' } }` | `tasks.register("hello") { doLast { println("Hi") } }` |
| **Typed Built-in Task** | `tasks.register('cleanDir', Delete) { delete 'out/' }` | `tasks.register<Delete>("cleanDir") { delete("out/") }` |
| **Custom Task Class** | `tasks.register('custom', MyTask) { value.set('x') }` | `tasks.register<MyTask>("custom") { value.set("x") }` |
| **Lazy Configuration** | `tasks.named('hello') { description = 'hi' }` | `tasks.named("hello") { description = "hi" }` |
| **Task Wiring (Lazy)** | `out.set(tasks.named('producer').flatMap { it.file })` | `out.set(tasks.named<Producer>("producer").flatMap { it.file })` |
| **Delegate Registration**| N/A | `val hello by tasks.registering { doLast { ... } }` |

---

## 2. Dynamic vs. Static Accessors

In **Groovy DSL**, you can reference tasks and configurations dynamically via their name strings directly:
```groovy
// Groovy: Dynamic lookup works natively
test.dependsOn myCustomTask
```

In **Kotlin DSL**, lookups must be statically typed. Kotlin DSL generates **type-safe accessors** for tasks and configurations applied via the `plugins {}` block.

```kotlin
// Kotlin: Type-safe accessors generated by plugins block
tasks.test {
    testLogging.showStackTraces = true
}
```

### When Type-Safe Accessors Are Unavailable
Type-safe accessors are **not** generated and cannot be used in the following contexts:
1. In precompiled script plugins (e.g., `build-logic/src/main/kotlin/my-convention.gradle.kts`).
2. Inside binary plugins (compiled classes).
3. If the plugin was applied using the legacy `apply(plugin = "...")` syntax.
4. For tasks registered dynamically in other scripts or plugins.

In these cases, you must fall back to the **string-based typed API**:
```kotlin
// Fallback syntax when accessors are unavailable
tasks.named<Test>("test") {
    testLogging.showStackTraces = true
}
```

## 3. Class Declaration Differences

Custom tasks written as classes also differ between the two languages:

### Groovy Custom Task
```groovy
import org.gradle.api.DefaultTask
import org.gradle.api.provider.Property
import org.gradle.api.tasks.Input
import org.gradle.api.tasks.TaskAction

abstract class GreetingTask extends DefaultTask {
    @Input
    abstract Property<String> getGreeting()

    @TaskAction
    void greet() {
        println greeting.get()
    }
}
```

### Kotlin Custom Task
```kotlin
import org.gradle.api.DefaultTask
import org.gradle.api.provider.Property
import org.gradle.api.tasks.Input
import org.gradle.api.tasks.TaskAction

abstract class GreetingTask : DefaultTask() {
    @get:Input
    abstract val greeting: Property<String>

    @TaskAction
    fun greet() {
        println(greeting.get())
    }
}
```

## 4. Groovy to KTS Migration Steps

1. **Double Quotes**: Replace single quotes `'` with double quotes `"` for all string literals.
2. **Method Parentheses**: Wrap function parameters in parentheses:
   - Groovy: `implementation 'org:lib:1.0'`
   - Kotlin: `implementation("org:lib:1.0")`
3. **Properties Assignment**: Use `=` for assignment instead of spaces or function-like calls:
   - Groovy: `group = 'verification'` or `group 'verification'`
   - Kotlin: `group = "verification"`
4. **Task Register Type**: Replace class arguments with generics:
   - Groovy: `tasks.register('checkBadging', CheckBadgingTask) { ... }`
   - Kotlin: `tasks.register<CheckBadgingTask>("checkBadging") { ... }`
5. **Provider Setters**: Use `.set()` or `=` (if using `import org.gradle.kotlin.dsl.assign` in Gradle 8+) for lazy properties:
   - Groovy: `goldenBadging.set(file)`
   - Kotlin: `goldenBadging.set(file)` or `goldenBadging = file`
